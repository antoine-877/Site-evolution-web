Laboratoire URL & HTTP


Protocole

Le protocole indique la façon dont le navigateur et le serveur échangent des informations.

Par exemple, http:// ou https://.

Le “s” dans https signifie que la connexion est sécurisée.

Domaine

Le domaine correspond au nom du site web que tu veux visiter.

C’est comme l’adresse principale d’une maison sur Internet.

Par exemple, www.exemple.com indique à ton navigateur sur quel serveur se rendre pour afficher le site.

Chemin

Le chemin indique la page ou la section précise du site que tu veux consulter.

Il se trouve après le nom de domaine, comme dans www.exemple.com/blog/article1.

Ici, le navigateur affiche la page “article1” du dossier “blog”.

Paramètres

Les paramètres contiennent des informations supplémentaires envoyées à la page pour personnaliser son affichage ou son contenu.

Ils apparaissent après un ? dans l’URL, comme dans www.exemple.com/recherche?mot=chat.

Ici, le paramètre mot=chat indique que l’utilisateur cherche le mot “chat”.

Ancre

L’ancre sert à indiquer une position précise à l’intérieur d’une page, comme un marque-page qui permet d’aller directement à une section spécifique.

Elle commence par un # dans l’URL, par exemple : www.exemple.com/guide#chapitre3.

Ici, le navigateur affiche directement la partie “chapitre 3” de la page.

Les codes d’état HTTP indiquent le résultat d’une requête entre le navigateur et le serveur. Chaque code donne une information sur ce qui s’est passé : succès, redirection, erreur du client ou du serveur.

Concepts clés

Principe

Un nom de domaine est l’adresse unique d’un site web, permettant de le trouver facilement.

Par exemple : www.exemple.com.

Il se compose généralement de plusieurs parties :

  • Sous-domaine : comme www, optionnel
  • Domaine principal : le nom propre, ex. exemple
  • Extension : comme .com, .fr

Les noms de domaine sont gérés par le DNS (Domain Name System), qui traduit le nom en adresse IP pour que le navigateur puisse trouver le serveur.

https:// www.exemple.com /page1 ?id=123 #section2
Dépôt / Réservation
Les noms de domaine sont réservés auprès de registraires pour garantir leur unicité.

Comment ça fonctionne :
  • Vous choisissez un nom disponible et vérifiez qu'il n'est pas déjà enregistré.

  • Vous payez le dépôt auprès d’un registraire officiel pour le réserver, généralement pour 1 an renouvelable.

  • Le registraire enregistre le nom dans le DNS pour qu’il soit accessible sur Internet.

Il est important de choisir un registraire fiable et reconnu, comme Gandi ou OVH.

Acteurs
Les principaux acteurs dans le monde des noms de domaine sont :
  • Registraires : sociétés accréditées qui permettent de réserver et gérer un nom de domaine.

  • Registres : organisations qui maintiennent la base officielle des noms de domaine d’une extension (.fr, .com, .org...).

  • Hébergeurs : fournissent l’espace serveur pour que le site web associé au domaine soit accessible sur Internet. En savoir plus.

  • Utilisateurs : les particuliers ou entreprises qui possèdent et utilisent le nom de domaine pour leur site web ou service en ligne.

Tous ces acteurs travaillent ensemble pour garantir que les noms de domaine restent uniques, disponibles et accessibles à tous sur Internet.

Cycle de vie
Un nom de domaine traverse plusieurs étapes tout au long de sa vie :
  • Actif : le domaine est enregistré et fonctionne normalement. En savoir plus.

  • Expiré : le domaine n’a pas été renouvelé avant la date d’expiration et devient indisponible.

  • Renouvellement / Grace period : période pendant laquelle le propriétaire peut encore récupérer son domaine après expiration.

  • Supprimé : le domaine est libéré et peut être enregistré par quelqu’un d’autre.

Comprendre le cycle de vie d’un domaine est essentiel pour éviter de perdre l’accès à son site web ou service en ligne.

DNS
Le DNS (Domain Name System) agit comme un annuaire pour Internet. Il traduit le nom de domaine que vous tapez dans le navigateur en adresse IP que les ordinateurs comprennent.
  • Par exemple, www.exemple.com peut correspondre à 192.168.1.1.

  • Sans DNS, vous devriez mémoriser les adresses IP de chaque site web que vous voulez visiter !

Standards du Web

Rôle

Les standards du Web définissent comment créer et interpréter les sites web.
Des organismes comme le W3C publient ces règles pour assurer que tous les navigateurs et outils web fonctionnent correctement.

Pourquoi

Sans standards, chaque navigateur pourrait afficher différemment un site, ce qui créerait le chaos.
Les standards garantissent que les pages web sont compatibles, accessibles et fiables pour tous les utilisateurs.

Exemple

Par exemple, le standard HTML et CSS définit comment écrire un bouton ou un formulaire.
Grâce à ces règles, un site s'affiche correctement sur Chrome, Firefox, Safari, et même sur mobile.

Sécurité du Web

Attaques courantes

Les sites web peuvent subir différentes attaques :

  • Phishing : tromper l'utilisateur pour voler ses informations (en savoir plus).

  • Malware : logiciels malveillants qui infectent les ordinateurs.

  • Brute Force : tenter toutes les combinaisons possibles pour deviner un mot de passe.

  • Scraping vs. abus : récupération automatique de données, parfois légale, parfois abusive.

Bonnes pratiques

Pour sécuriser un site et ses utilisateurs :

  • Utiliser HTTPS pour chiffrer les échanges.

  • Mettre en place des mots de passe forts et authentification multi-facteurs.

  • Mettre à jour régulièrement logiciels et plugins.

  • Limiter les tentatives de connexion et protéger contre le scraping abusif.

Exemple pratique

Par exemple, Gmail bloque automatiquement les tentatives de connexion suspectes et signale le phishing.

Tandis que les sites sérieux utilisent HTTPS et alertent l’utilisateur en cas d’abus .
